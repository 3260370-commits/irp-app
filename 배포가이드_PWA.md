# IRP 앱(PWA) 배포 가이드 — GitHub Pages

로컬 HTML 파일 방식을 **주소로 접속하는 설치형 앱**으로 전환하는 절차입니다.
전환 후에는 PC·폰 모두 같은 주소 하나만 쓰므로 파일 관리(경로 고정, 덮어쓰기)가 아예 필요 없어집니다.

## 구조 (보안 원칙 — 참조 문서 4-2 준수)

| 저장소 | 공개 여부 | 내용 |
|---|---|---|
| `irp-app` (신규) | **공개** | 앱 코드만 (index.html, manifest, sw.js, 아이콘) — 실제 데이터·토큰 없음 |
| `irp-monitor` (기존) | **비공개** | holdings.json (실제 데이터) — 그대로 유지 |

토큰은 각 기기 브라우저(localStorage)에만 저장되므로 앱 주소가 공개되어도 남이 내 데이터를 볼 수 없습니다.

## 1단계. 공개 저장소 생성 및 파일 업로드

1. GitHub → New repository → 이름 `irp-app` → **Public** → Create
2. 이 폴더의 5개 파일을 저장소에 드래그 앤 드롭 → Commit
   - `index.html`, `manifest.webmanifest`, `sw.js`, `icon-192.png`, `icon-512.png`
   - (배포가이드_PWA.md는 올릴 필요 없음)

## 2단계. GitHub Pages 켜기

1. irp-app 저장소 → **Settings → Pages**
2. Source: **Deploy from a branch** / Branch: **main** / 폴더: **/ (root)** → Save
3. 1~2분 뒤 상단에 주소가 표시됨: `https://3260370-commits.github.io/irp-app/`

## 3단계. PC에 앱 설치

1. Chrome으로 위 주소 접속
2. 주소창 오른쪽 **설치 아이콘(모니터+↓)** 클릭 → 설치
   - 바탕화면·시작메뉴에 IRP 아이콘 생성, 독립 창으로 실행됨 (이전에 안 되던 바로가기 문제가 이걸로 해결됩니다)
3. ⚙ 설정: `3260370-commits` / `irp-monitor` / `holdings.json` / `main` / 토큰 입력 → 저장
   - https 주소는 새 저장공간이라 **설정을 한 번 다시 입력**해야 합니다. 저장하면 자동으로 GitHub 데이터를 가져옵니다 (25,541,112원 확인)

## 4단계. 폰에 앱 설치 (Android)

1. 폰 Chrome으로 같은 주소 접속
2. 메뉴 ⋮ → **앱 설치** (또는 "홈 화면에 추가") → 홈화면에 IRP 아이콘 생성
3. ⚙ 설정에 동일하게 입력 (토큰은 폰에서 다시 입력) → 자동으로 데이터 표시

## 자동 동기화 동작 방식

- **앱을 열 때 / 화면에 다시 돌아올 때**: GitHub 최신 데이터를 자동으로 가져옴
- **수정 후 4초 뒤**: 자동으로 GitHub에 커밋 (연속 수정은 마지막 것만)
- **충돌 시** (양쪽에서 동시에 수정): 어느 쪽을 남길지 확인창으로 물어봄
- 하단 상태줄에서 "✅ 자동 커밋 완료 / ✅ 최신 상태" 메시지로 확인 가능
- 수동 버튼(가져오기/올리기)도 그대로 사용 가능, ⚙ 설정에서 자동 동기화 끄기도 가능

## 전환 후 정리

- 기존 로컬 HTML 파일(`C:\00- USER\O-claude\7-IRP\...html`)은 더 이상 쓰지 않음 (혼용하면 데이터 꼬임의 원인이 되므로 삭제 권장)
- 데이터는 이미 irp-monitor에 커밋되어 있으므로 새 앱에서 자동으로 불러와짐 — 별도 이전 작업 불필요
- 앱 코드를 수정할 일이 있으면 irp-app 저장소의 index.html만 교체하면 모든 기기에 자동 반영됨
